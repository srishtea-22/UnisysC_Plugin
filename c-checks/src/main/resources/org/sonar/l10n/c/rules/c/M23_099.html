Why is this an issue?

The unary minus operator - negates a numeric value by performing arithmetic negation. When applied to an unsigned integer type, the behavior can be surprising and error-prone due to the semantics of unsigned arithmetic.

For example, when you negate an unsigned value, the result wraps around according to unsigned arithmetic rules. A value like -10U does not represent negative ten; instead, it underflows and produces a large positive unsigned value. This counterintuitive behavior often indicates a logic error or type confusion. 

unsigned int ui = 10U;
int result = -ui;  // Noncompliant: negating unsigned produces unexpected result

Such code typically signals that the developer either:

  • Intended to use a signed type but mistakenly declared an unsigned type
  • Misunderstands how unsigned arithmetic works
  • Is attempting an operation that does not make semantic sense for unsigned values

How to fix it

Ensure that negation is only applied to signed types. If you need to negate a value, declare it as a signed type:

/* Recommended: Use a signed type */
int si = 10;
int result = -si;  // OK: negation of signed integer

/* Or explicitly cast if unsigned is required for other reasons */
unsigned int ui = 10U;
int result = -(int)ui;  // More explicit about the type conversion
