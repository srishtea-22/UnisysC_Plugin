Why is this an issue?

In C, a function declared with a non-void return type must return a value on every possible execution path. If any path through the function reaches the closing brace without a return statement, the behavior is undefined — the caller receives a garbage value, which can cause crashes or silent data corruption.

Exceptions

  • Functions with return type void are ignored.
  • Functions with an empty body are ignored (handled by a separate rule).

How to fix it

Ensure every branch of your function ends with a return statement that provides a value.

Code examples

Noncompliant code example

int getSign(int x)        /* Noncompliant — else path missing return */
{
    if (x > 0) {
        return 1;
    } else if (x < 0) {
        return -1;
    }
                          /* missing return when x == 0 */
}

Compliant solution

int getSign(int x)
{
    if (x > 0) {
        return 1;
    } else if (x < 0) {
        return -1;
    }
    return 0;             /* all paths now covered */
}
