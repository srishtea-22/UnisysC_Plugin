Why is this an issue?

The switch statement is a powerful control flow mechanism, but when dealing with only a few simple conditions, an if statement is often clearer and more maintainable.

Using switch for trivial cases adds unnecessary complexity and cognitive load. When a switch statement has very few branches (especially just 1-2 cases), an if or if-else statement would be simpler and more readable.

Example:

    int value = getValue();
    
    // Noncompliant - unnecessary switch for a simple case
    switch (value) {
        case 0:
            doSomething();
            break;
        default:
            doSomethingElse();
            break;
    }
    
    // Compliant - simpler if statement
    if (value == 0) {
        doSomething();
    } else {
        doSomethingElse();
    }
    
    // Compliant - multiple complex branches justify switch
    switch (status) {
        case ACTIVE:
            handleActive();
            break;
        case INACTIVE:
            handleInactive();
            break;
        case PENDING:
            handlePending();
            break;
        default:
            handleUnknown();
            break;
    }
