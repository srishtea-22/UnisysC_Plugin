Why is this an issue?

Program-terminating functions such as exit(), abort(), _Exit(), and quick_exit() abruptly terminate program execution without proper cleanup. This can lead to resource leaks, data corruption, and unpredictable behavior.

Using these functions makes code difficult to test, maintain, and integrate into larger systems. Instead, functions should return error codes or use exception-like mechanisms to allow the caller to handle errors gracefully.

Example:

    #include <stdlib.h>
    
    int processFile(const char *filename) {
        FILE *f = fopen(filename, "r");
        if (f == NULL) {
            exit(1);            // Noncompliant - abrupt termination
        }
        // process file
        fclose(f);
        return 0;
    }
    
    int processFileCompliant(const char *filename) {
        FILE *f = fopen(filename, "r");
        if (f == NULL) {
            return -1;          // OK - return error code
        }
        // process file
        fclose(f);
        return 0;
    }
