Why is this an issue?

Subtracting pointers that do not address elements of the same array is undefined behavior. This can lead to unpredictable results, memory corruption, and security vulnerabilities.

Pointer arithmetic is only well-defined when both pointers point to elements within the same array or to one element past the end of the array. Subtracting pointers from different arrays or incompatible memory regions violates the C standard and results in undefined behavior.

Example:

    int arr1[10];
    int arr2[10];
    int *p1 = &arr1[5];
    int *p2 = &arr2[3];
    
    int diff = p1 - p2;           // Noncompliant - pointers from different arrays
    
    int diff2 = &arr1[7] - &arr1[2];  // OK - both point to same array
    int diff3 = p1 - &arr1[0];   // OK - both point to same array
