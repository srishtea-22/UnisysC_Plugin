Why is this an issue?

Casting a pointer type to an integral type is problematic for several reasons:

  • Breaks Code Analysis: Pointer tracking tools become unreliable when pointers are cast to integers, making it harder for static analysis tools to understand program behavior.
  • Platform Portability: The size of a pointer and the size of integral types vary across different platforms. A cast that works on one platform may fail or produce unexpected results on another.
  • Undefined Behavior: Casting pointers to integral types (other than pointer-sized types) can result in undefined behavior, leading to runtime errors or data corruption.
  • Readability: Such casts indicate low-level manipulation that is often not immediately clear to other developers reviewing the code.
  • Maintenance Issues: Code using pointer-to-integral casts is difficult to maintain and refactor safely.

While casting between pointers and integers may be unavoidable when addressing memory-mapped registers or other hardware-specific features, such cases should be handled with explicit types designed for this purpose. 

void test() {
    int *p = malloc(sizeof(int));
    
    int id = (int) p;          // Noncompliant: pointer cast to integral
    long addr = (long) p;      // Noncompliant: pointer cast to integral
}

How to fix it

Option 1: Avoid the cast altogether

If you don't actually need an integral representation, simply pass the pointer directly:

void process(int *data) {
    /* Use the pointer directly */
}

int main() {
    int *ptr = malloc(sizeof(int));
    process(ptr);  /* Pass pointer, no cast needed */
    return 0;
}

Option 2: Use platform-safe pointer types

If you must convert a pointer to an integral type (e.g., for memory-mapped hardware), use uintptr_t or intptr_t which are guaranteed to hold any pointer value:

#include <stdint.h>

void test() {
    int *p = malloc(sizeof(int));
    
    uintptr_t addr = (uintptr_t) p;  /* Compliant: safe conversion */
    intptr_t signed_addr = (intptr_t) p;  /* Also compliant */
}

Option 3: Use a struct to encapsulate pointer metadata

typedef struct {
    void *ptr;
    unsigned long handle;
} PointerHandle;

PointerHandle createHandle(void *ptr, unsigned long id) {
    PointerHandle h = { ptr, id };
    return h;  /* Store id separately, not by casting */
}

By avoiding pointer-to-integral casts (or using safe types when necessary), your code becomes more portable, maintainable, and amenable to static analysis.
