Why is this an issue?

An empty function is generally considered bad practice and can lead to confusion, readability, and maintenance issues. Empty functions bring no functionality and are misleading to others as they might think the function implementation fulfills a specific and identified requirement.

There are several reasons for a function not to have a body:

  • It is an unintentional omission, and should be fixed to prevent unexpected behavior in production.
  • It is not yet, or never will be, supported. In this case a comment should explain this clearly.
  • The function is an intentionally blank stub. In this case a nested comment should explain the reason.

How to fix it

Code examples

Noncompliant code example

void shouldNotBeEmpty() {    /* Noncompliant - function is empty */
}

void notImplemented() {      /* Noncompliant - function is empty */
}

void emptyOnPurpose() {      /* Noncompliant - function is empty */
}

Compliant solution

void shouldNotBeEmpty() {
    doSomething();
}

void notImplemented() {
    /* Not yet implemented — planned for version 2.0 */
}

void emptyOnPurpose() {
    /* Intentionally left empty — overridden by platform layer */
}
