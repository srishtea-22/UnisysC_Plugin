Why is this an issue?

Each level of pointer indirection (each *) adds a layer of complexity and mental overhead for the developer. High levels of indirection make code significantly harder to read, track, and debug.

Following the "Two Levels of Indirection" rule ensures that data structures remain manageable. Code like ***ptr requires three separate memory lookups to access the underlying value, which can also impact performance due to cache misses.

int ***data;  // Noncompliant

How to fix it

If you require more than two levels of indirection, consider refactoring your data using struct to encapsulate intermediate layers. This provides meaningful names to the levels of data you are accessing.

/* Recommended Refactoring */
struct Row {
  int *cells;
};

struct Matrix {
  struct Row *rows;
};

struct Matrix *myMatrix; // Only one level of indirection to access the Matrix structure.
